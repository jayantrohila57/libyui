# Tooling

LibyUI comes with an opinionated but configurable developer toolchain. This section covers the CLI for scaffolding components, TypeScript setup, and the Biome linter/formatter configuration used in the project.

## Setup overview

<Steps>
  <Step>Install LibyUI and its peer dependencies via your package manager</Step>
  <Step>Run `pnpm shadcn` to initialize the component registry and generate your `components.json`</Step>
  <Step>Configure TypeScript path aliases (`@/`) in `tsconfig.json`</Step>
  <Step>Add Biome (or your preferred linter) for consistent formatting across the project</Step>
</Steps>

## Common questions

<Accordion type="single" collapsible className="w-full">
  <AccordionItem value="item-1">
    <AccordionTrigger>Do I need the CLI?</AccordionTrigger>
    <AccordionContent>
      No — you can copy component source files manually. The CLI (`pnpm shadcn`) just automates the process of adding components to your project and keeping them up to date.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="item-2">
    <AccordionTrigger>Do I have to use Biome?</AccordionTrigger>
    <AccordionContent>
      No. LibyUI uses Biome internally for speed and zero-config formatting. You can use ESLint and Prettier instead — nothing in LibyUI depends on your linter choice at runtime.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="item-3">
    <AccordionTrigger>Can I use LibyUI without TypeScript?</AccordionTrigger>
    <AccordionContent>
      Yes, but you'll lose prop autocompletion and type safety for variants and sizes. TypeScript is strongly recommended for the best developer experience.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="item-4">
    <AccordionTrigger>What version of Tailwind CSS is required?</AccordionTrigger>
    <AccordionContent>
      LibyUI requires Tailwind CSS v4. It uses the PostCSS plugin (`@tailwindcss/postcss`) and the `tw-animate-css` package for animations. Make sure your `postcss.config.mjs` is configured accordingly.
    </AccordionContent>
  </AccordionItem>
</Accordion>
