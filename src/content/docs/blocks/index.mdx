# Blocks

Blocks are pre-built, reusable layout sections that combine multiple components into cohesive, production-ready parts of your UI. They follow best practices for design, accessibility, and responsiveness — so you can ship faster without starting from scratch.

## How blocks work

<Steps>
  <Step>Browse the block category that matches your need</Step>
  <Step>Copy the block code into your project</Step>
  <Step>Customize colors, copy, and layout using Tailwind CSS and props</Step>
</Steps>

## Common questions

<Accordions>
  <Accordion title="Are blocks just components?">
    Blocks are compositions of multiple components, not standalone primitives. A Hero block, for example, might use `Button`, `Card`, and typography elements together to form a complete page section.
  </Accordion>
  <Accordion title="Can I use only part of a block?">
    Yes. Blocks are not packaged exports — they are code you copy into your project. Feel free to extract only the parts you need and adjust the markup freely.
  </Accordion>
  <Accordion title="Are blocks responsive?">
    All blocks are mobile-first. They are designed to look great at every breakpoint out of the box, though you can easily adjust breakpoints via Tailwind utilities.
  </Accordion>
  <Accordion title="Do blocks follow accessibility standards?">
    Yes. Blocks use semantic HTML and LibyUI's accessible primitive components. Navigation blocks use `<nav>`, form blocks use proper `<label>` associations, and so on.
  </Accordion>
</Accordions>
