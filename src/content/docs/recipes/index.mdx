# Recipes

Recipes are step-by-step, copy-paste-ready guides for building real-world features with LibyUI. Each recipe walks you through a concrete goal — from start to finished, working code.

<Accordion type="single" className="w-full">
  <AccordionItem value="item-1">
    <AccordionTrigger>What is a recipe?</AccordionTrigger>
    <AccordionContent>
      A recipe is an end-to-end walkthrough for a specific UI task. It starts from a clear goal ("build a delete confirmation modal"), lists the components you'll need, and provides complete, working code you can drop into your project.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="item-2">
    <AccordionTrigger>Recipes vs. Patterns</AccordionTrigger>
    <AccordionContent>
      Recipes are concrete and opinionated — they show one good way to solve a specific problem. Patterns are conceptual and explore the trade-offs across multiple approaches. Use recipes when you want something working fast; use patterns when you're deciding on an approach.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="item-3">
    <AccordionTrigger>Can I request a recipe?</AccordionTrigger>
    <AccordionContent>
      Yes. Open an issue on GitHub with the title "Recipe Request: [your topic]" and describe what you're trying to build. High-demand requests get prioritized.
    </AccordionContent>
  </AccordionItem>
</Accordion>
